Bootstrap 5 – Directory Structure

Exploring the folders inside the Bootstrap 5 download package.

Downloaded Bootstrap Folder Layout

bootstrap/
├── css/
│   ├── bootstrap.css               ← Full unminified CSS
│   ├── bootstrap.min.css           ← Minified CSS (use in production)
│   ├── bootstrap.rtl.css           ← Right-to-left language support
│   ├── bootstrap-grid.css          ← Grid system only
│   ├── bootstrap-reboot.css        ← CSS reset/normalize only
│   └── bootstrap-utilities.css     ← Utility classes only
│
├── js/
│   ├── bootstrap.js                ← Full JS (requires Popper.js separately)
│   ├── bootstrap.min.js            ← Minified JS
│   ├── bootstrap.bundle.js         ← JS bundled WITH Popper.js ✅ (recommended)
│   ├── bootstrap.bundle.min.js     ← Minified bundle (use in production) ✅
│   └── bootstrap.esm.js            ← ES Module version (for modern JS imports)
📁 css/ folder

Contains all compiled CSS files. Use bootstrap.min.css for production. Specialized builds like bootstrap-grid.css let you include only what you need.

📁 js/ folder

Contains JavaScript plugins for modals, dropdowns, tooltips, etc. The bootstrap.bundle.min.js file includes Popper.js — the easiest option for most projects.

📁 icons/ (separate)

Bootstrap Icons is a separate package. Install via: npm install bootstrap-icons or use the CDN link in your HTML.

💡 Tip: For most projects, you only need two files: bootstrap.min.css and bootstrap.bundle.min.js.
